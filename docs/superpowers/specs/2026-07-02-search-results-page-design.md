# 搜索结果页美化设计

## 目标

将搜索结果页优化为“液体/玻璃感的搜索入口 + 首页式清爽结果卡片”的视觉组合，延续首页顶部导航的灵动气质，同时保持搜索页的信息效率和可读性。

## 范围

- 优化 `src/views/search/SearchView.vue` 的页面结构与语义化 class。
- 优化 `src/assets/css/search.scss` 的搜索页样式、响应式布局和暗色模式。
- 不改动搜索接口、分页加载逻辑、排序逻辑和路由行为。
- 不引入新的运行时依赖。

## 视觉方向

搜索页顶部保留现有 banner 图，但内容层次更精致：标题、统计信息和搜索状态以更轻的排版呈现。主体搜索区使用半透明、细边框、柔和阴影和轻微渐变，形成接近玻璃/液体的主操作区域。结果列表采用首页文章卡片的清爽风格，弱化旧式浮动布局带来的割裂感。

## 页面结构

- Banner：展示页面标题和最近一次搜索统计。
- 搜索控制区：包含类型选择、关键词输入、搜索图标、每日热搜和文章排序。
- 文章结果区：使用卡片式横向结果，包含封面、作者、标题、摘要、分类、标签和统计数据。
- 用户结果区：使用轻量用户名片，突出头像、昵称、性别和简介。
- 加载与空状态：继续沿用现有 `LoadMore` 行为。

## 响应式与暗色模式

- 桌面端保持结果卡片横向布局，搜索控制区居中并适度拉宽。
- 移动端改为单列卡片，热搜词横向自然换行，结果信息避免拥挤。
- 暗色模式使用深色玻璃面板、低饱和边框和浅色文字，避免纯黑块和高对比刺眼效果。

## 验证

- 运行现有单元测试，确保页面结构调整没有破坏基础组件。
- 运行类型检查或构建检查，确保 Vue 模板和 TypeScript 仍可通过。
- 若启动本地页面检查，重点查看桌面端、移动端和暗色模式下搜索框、热搜词、文章卡片、用户卡片是否有重叠或溢出。
